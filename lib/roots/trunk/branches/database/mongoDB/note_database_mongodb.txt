NoSQL è un'alternativa al database tradizionale ed è un approccio più conveniente.
 Qui i dati possono facilmente crescere distribuendosi su molti server cloud ordinari ed economici.

Inoltre, il database NoSQL è esattamente il tipo di database in grado di gestire il tipo di dati non strutturati,
 irregolari e imprevedibili (ad es. dati utente/tweet) che soddisfa i requisiti aziendali.

Tuttavia, la GRANDE domanda è: dove applichi NoSQL?
La semplice risposta è: applicalo in applicazioni/strumenti/sito Web in cui è necessario gestire un volume elevato 
di dati con prestazioni elevate e disponibilità elevata.
 In base alle tue esigenze, puoi optare per uno qualsiasi dei 4 tipi base di database NoSQL.

3. MongoDB
MongoDB è un database distribuito generico creato per gli sviluppatori di applicazioni di questa generazione
 da utilizzare nel cloud.Questo è un database di documenti che archivia i dati in documenti simili a JSON 
 che è molto più potente ed efficiente rispetto ai tradizionali database di righe e colonne. 
 MongoDB supporta anche vari metodi di ricerca come la ricerca geografica, la ricerca di testo, 
 la ricerca di grafici, ecc. Un altro vantaggio di MongoDB è che fornisce sicurezza di prima classe per i suoi client inclusi SSL, firewall, crittografia, ecc.
  E la cosa migliore sei tu può anche creare visualizzazioni utilizzando i dati MongoDB
   e connettersi con qualsiasi strumento di Business Intelligence compatibile con il protocollo MySQL.
